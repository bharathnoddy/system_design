# Bluesky

> Decentralized social via AT Protocol — federation, DID, PDS, AppView

## What to Study

- AT Protocol architecture: how identity, data, and indexing are separated
- Personal Data Servers (PDS) as user-controlled data homes
- Federated relay (firehose) and how AppViews consume and index the event stream
- Decentralized Identifiers (DID) and key rotation for portable identity
- Lexicon schemas as a typed RPC and data definition layer

## Why This Matters for Platform Architecture

Bluesky demonstrates an alternative to centralized social graphs — splitting identity, storage, and view computation into independently operable components. It is a live case study in federated system design where protocol decisions directly constrain what applications can build.
